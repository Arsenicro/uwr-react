# Task: Recipe Book - Zustand (10p)

Your task is to create (or Rewrite) a recipe search engine in React.js. **This task involves creating the same application as in the previous lab, but using the Zustand library to compare both approaches.** The main educational goal of this task is to practice managing more complex application state using the **Zustand** library.

## Materials

- [Zustand Documentation](https://zustand.docs.pmnd.rs/learn/getting-started/introduction)

## Environment

Create a new project using Vite: `npm create vite@latest my-recipe-book-zustand -- --template react-ts`.
Install Zustand library: `npm install zustand`.

## Requirements

### 1. Implementation

Design a "Recipe Book" application that allows for:

- **Displaying recipes**: A list of tiles/elements, each containing the title and content of the recipe.
- **Adding recipes**: A simple form allowing the addition of a new recipe to the list (title + content).
- **Managing recipes**:
  - Deleting a selected recipe.
  - Marking a recipe as "favorite" (this should be visually apparent).
- **Filtering**:
  - Option to display only "favorite" recipes (e.g., checkbox or toggle).
  - Searching for recipes by keyword (text input searching both the name and content).
- **Styles**: The application should have basic, aesthetic styles (defined e.g., in `App.css`).

### 2. Architecture

- **Zustand**:
  - All recipe state management logic must be moved to a Zustand store (e.g., in `store.ts` or `useRecipeStore.ts`).
  - The store should expose the state (list of recipes, optional filters) and actions (add, remove, toggle favorite).
  - Components should simply use the hook generated by `create` from the Zustand library.
- **Components**: Divide the application into smaller, readable components.
- **TypeScript**: The code must be fully typed (no `any`).
